Třetí verzi webu Tanečního studia jsem chtěl využít k celkové modernizaci jak vizuální identity, tak celé architektury aplikace.

Nový grafický design by měl celou vizuální identitu posunout na modernější styl a zjednodušit ji, jelikož ta předchozí byla problematická při tisk na např. propagační materiály.

Nová architektura odděluje front-end a back-end; front-end je vytvořen s pomocí <a href="https://gatsbyjs.com" target="_blank" rel="noopener noreferrer">Gatsby</a> a komunikuje s API na PHP <a href="https://slimframework.com" target="_blank" rel="noopener noreferrer">Slim</a> back-endu pomocí GraphQL. Toto oddělení mne nutí dodržovat správné způsoby návrhu API a zároveň poskytuje sílu frameworku <a href="https://reactjs.org" target="_blank" rel="noopener noreferrer">React</a> a balíčků ekosystému <a href="https://npmjs.com" target="_blank" rel="noopener noreferrer">NPM</a>.

Nově oproti druhé verzi poskytuje šifrování osobních údajů, robustní architekturu a cache vybraných požadavků, což dohromady zlepšuje bezpečnost i výkon webvé aplikace.

V průběhu vývojé této verze se objevila perspektiva poskytuntí této rozsáhlé webové aplikace dalším klubům, proto prošel zdrojový kód řadou úprav pro jednoduchou přizpůsobitelnost a platforma dostala název **MoveS**.
